Al realizar una petición GET a la ruta http://localhost:3000/reportes, lo que sucede es que el servidor Express responde devolviendo el array llamado "reportes" en formato JSON. Si es la primera vez que se consulta antes de haber enviado datos, el servidor devuelve una lista vacía representada por corchetes []. Sin embargo, luego de ejecutar peticiones POST para registrar incidentes, la ruta GET devuelve la lista completa con todos los objetos de reportes acumulados en la memoria, mostrando de forma estructurada el ID asignado automáticamente, el tipo de incidencia y la descripción de cada una.
